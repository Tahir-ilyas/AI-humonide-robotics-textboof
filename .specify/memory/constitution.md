# AI Humanoid Robotics Textbook Constitution

## Purpose
Guide AI agents to create a clear, accurate technical book using Spec-Kit Plus, Docusaurus, Claude Code, and deploy it on GitHub Pages.

## Core Principles

### I. Content Quality
Content MUST be concise, correct, and beginner-friendly.

### II. Accuracy and Truthfulness
The book MUST be free of hallucinations, fictional commands, or misleading information.

### III. Format Adherence
All content MUST strictly follow Spec-Kit Plus formats.

### IV. Consistency
The book MUST maintain consistent style, terminology, and structural elements throughout.

### V. Chapter Structure
Each chapter MUST adhere to the following structure: Title, Overview, Objectives, Main Content, Examples, Summary.

### VI. Specification-Driven Content Generation
Specifications MUST be processed into chapters.

### VII. Docusaurus Configuration
The Docusaurus configuration and sidebar MUST be valid and functional.

### VIII. Code Reproducibility
All code examples MUST be correct and reproducible.

### IX. Operational Readiness
Navigation and deployment to GitHub Pages MUST function correctly.

## Agent Behavior

### I. Constitution Adherence
Agents MUST follow this constitution exactly.

### II. Objectivity and Correctness
Agents MUST prioritize correctness over creativity.

### III. Efficiency in Clarification
Agents MUST ask only necessary clarifying questions.

## Deliverable
A complete, deployable Docusaurus book generated using Spec-Kit Plus.

## Governance
This constitution supersedes all other project practices. Amendments require documentation, approval, and a migration plan. All PRs/reviews must verify compliance. Complexity must be justified.
